## 🚀 セットアップ(超シンプル)

### 1. UserScripts拡張をインストール(初回のみ)
- App Store で **「UserScripts」**(作者: Justin Wasack)を無料インストール
- iOS設定 > Safari > 機能拡張 > UserScripts を ON
- すべてのWebサイトで「許可」

### 2. PB-CART をインストール
- スマホ Safari で Netlify URL を開く(例: `https://xxx.netlify.app/`)
- リンク **「⬇ PB-CART をインストール」** をタップ
- UserScripts拡張のインストールダイアログ → **Save**

### 3. 設定する
- スマホ Safari でプレバンの**任意のページ**を開く(例: `https://p-bandai.jp/`)
- 起動バナー(画面上部)が出ることを確認
- **右下「⚙ 設定」** をタップ → 設定モーダルが全画面で開く
- 「＋ 商品を追加」 → URL貼付 → **「📥 商品名・時刻を取得」** タップ
- 同origin fetch で商品名・発売時刻が**自動入力**される(文字化けなし、地域判定なし)
- 種別/個数/通知等を調整 → 「💾 保存して閉じる」

### 4. 動作確認
- 設定した商品ページを Safari で開く
- 起動バナーに「🎯 ターゲット検出: 商品名」と表示
- 右下UIが「🎯 監視中」「⏰ 発売待ち」等で動作

## 🎛 主要機能

### 自動取得(同origin fetch)
- 設定モーダル内「📥 商品名・時刻を取得」ボタン
- p-bandai.jp 上で動くので CORS 制約なし
- h1 → og:title → title の順で抽出、フィルタ通過したもの
- 「予約受付開始 YYYY年M月D日 H時」を正規表現で抽出

### 投入処理
- 発売時刻まで自動待機
- 試行 10回 → リロード(最大5回)
- 成功 → カート確保後の挙動を選択(完全停止/超控えめ続行/通常)

### アクセス制限対応
- 連続検知で Discord警告 + 自動nuke(カート空時のみ)
- カート確保後の検知 → 全停止(WAPID再発行で既存カートが消えるリスク回避)

### キープアライブ
- ダミー商品で 50分毎にカート保持時間60分復活
- 推奨間隔45-58分(高頻度はアクセス制限リスク)

### 通知
- Discord Webhook (HIROさん指定 URL がデフォルト埋め込み)
- カート確保時にリッチEmbedで通知

### リアルタイムログ
- 駿河屋ボット風、時刻ミリ秒精度で全イベント記録
- 右下UI「📋 ログ」で展開、最大200行

## 🔧 トラブルシューティング

### 起動バナーが出ない
= UserScriptが動いていない
- UserScripts拡張で `PB-CART (プレバンカート支援)` がインストール済みか確認
- スクリプトが ON になっているか確認
- Safari「あ々」→ UserScripts → リスト確認

### 「⏸ このページは登録商品ではありません」と出る
= URL一致しない
- 設定で登録した URL と現在のURLを比較
- URLは `https://p-bandai.jp/item/item-XXXXXXXXXX/` 形式
- 末尾スラッシュの有無で判定するので、コピペ時に気をつける

### 「📥 取得」が失敗する
- ログを「📋 ログ」で確認
- アクセス制限を食らっている可能性 → 数分待つ or 強制リセット

### ログを見て報告
何か止まっている時は右下「📋 ログ」展開 → 内容確認 → スクリーンショットで報告

## ⚠ 既知の制約

- **iOS Safari + UserScripts拡張前提**(Tampermonkey互換)
- アクセス制限が継続した場合、Cookie全削除 = カートも消える
- カート確保後にアクセス制限を食らうと既存カートも消えるリスクあり(設計で全停止対応)
